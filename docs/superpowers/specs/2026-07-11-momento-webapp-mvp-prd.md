## 1. 개요

**Momento**는 1인·프리랜서를 위한 미니멀 프로젝트 관리 웹앱이다.
여러 프로젝트를 관리하되, 화면은 늘 **‘지금 집중할 단 하나’**를 건네 압도감을 없앤다.

- **한 줄 정의**: 여러 프로젝트를 다루지만, 지금의 답은 언제나 하나인 조용한 작업 공간
- **타깃**: 여러 건을 동시에 굴리는 1인 사업자·프리랜서·개인 생산성 사용자
- **경쟁 대비**: Jira/Asana의 협업·기능 과잉 ✗ → 개인 몰입·절제 ✓

### 해결하는 문제
혼자 여러 프로젝트를 굴리는 사람의 진짜 고통은 ‘할 일이 많다’보다 **‘도구가 나를 재촉하고 압도한다’**이다. 밀린 항목의 붉은 배지, 넘치는 목록, 끊기는 몰입이 문제다.

---

## 2. 목표 & 비목표

### 목표 (Goals)
1. 여러 프로젝트를 가로질러 **지금 할 하나**를 명확히 제시한다.
2. 떠오른 생각을 **흐름을 끊지 않고** 담게 한다 (빠른 메모).
3. 못 끝낸 일이 **압박이 아니라 조용한 이월**로 이어지게 한다.
4. 모든 화면이 브랜드 철학(여백·절제·재촉하지 않음)을 체감시킨다.

### 비목표 (Non-Goals, MVP 제외)
- 팀 협업(멤버 초대·댓글·할당·공유)
- 푸시/이메일 알림, 리마인더 스팸
- 화면·웹 스크린샷 캡처, 파일/이미지 클리핑 (텍스트 메모만)
- 간트·타임트래킹·리포트 대시보드
- 네이티브 모바일 앱 (반응형 웹까지만)
- 서드파티 연동(캘린더·슬랙 등)

## 3. 핵심 기능 (MVP)

### 3.1 지금 하나 (One Now) — 킬러 피처
- ‘오늘’ 화면 상단에 **단 하나의 태스크**를 크게(테라코타 포인트) 표시.
- **선정 방식(확정): 시간순.** 오늘 태스크는 시간순(오늘 목록에 담긴/예정된 순서)으로 정렬되고, 아직 끝내지 않은 것 중 **가장 앞선 하나**가 자동으로 ‘지금 하나’가 된다. 별도 수동 지정 없음.
- 액션: **[✓ 끝냈어요]** → 완료 처리 후 시간순 다음 태스크가 지금 하나로 승격 · **[다음 하나 →]** → 지금 것을 시간순 뒤로 보내고 다음 것을 앞세움(미루기).
- 나머지 오늘 태스크는 하단 ‘이어서’에 담백하게(작게) 나열.

### 3.2 다중 프로젝트 관리
- 프로젝트 CRUD, 각 프로젝트에 **식별 색상**(테라코타/세이지/블루/플럼 등 뮤트 팔레트) 지정.
- 사이드바에 프로젝트 목록 + 남은 태스크 수.
- **전체 보기**: 프로젝트 카드 그리드, 카드마다 **‘다음 하나’**와 남은 개수만 노출(전체 목록 나열 X).
- 태스크는 반드시 하나의 프로젝트에 속함. 오늘 화면은 모든 프로젝트를 가로지름.

### 3.3 빠른 메모 (Quick Note) — 인박스
- 전역 단축키 **⌘N**(웹: Cmd/Ctrl+N 대체 키 확정 필요)로 팝오버 → 한 줄 입력 → 저장 후 하던 일로 복귀.
- 프로젝트 지정·분류 없이 **메모함(Inbox)**에 모임.
- 메모함에서 각 항목을 나중에 프로젝트 태스크로 전환(triage)하거나 삭제.
- 목적: 몰입 보호 + 머릿속 비우기. (※ “캡처”는 스크린샷 오해가 있어 **‘빠른 메모/메모함’**으로 명명 확정.)

### 3.4 다정한 이월 (Gentle Rollover)
- 오늘 못 끝낸 태스크는 다음 날 오늘로 **조용히 이월**. 붉은 배지·“N일 지연”·연체 카운트 **없음**.
- 표시는 담백하게(예: “내일로 미룸” 라벨). 압박 문구 금지.

### 3.5 (선택) 하루의 마무리 (Daily Close) — v1.1 후보
- 저녁에 그날 끝낸 것만 담담히 보여주는 회고 화면. MVP 필수 아님.

## 4. 주요 화면 & 흐름

| 화면 | 목적 | 핵심 요소 |
|---|---|---|
| **오늘** | 지금 하나에 몰입 | 지금 하나 카드, 이어서 목록, 주간 완료 수 |
| **전체 보기** | 프로젝트 조망 | 프로젝트 카드(다음 하나 + 남은 수) |
| **프로젝트 상세** | 한 프로젝트의 태스크 관리 | 태스크 목록(우선순위 정렬), 추가/완료/이월 |
| **메모함** | 빠른 메모 정리 | 미분류 메모 목록 → 프로젝트로 전환 |
| **빠른 메모 팝오버** | 마찰 없는 입력 | 한 줄 입력 + 저장 |

### 대표 흐름
1. **몰입 흐름**: 오늘 화면 → 지금 하나 수행 → ✓ 끝냈어요 → 다음 하나 자동 노출.
2. **메모 흐름**: 작업 중 아이디어 → ⌘N → 한 줄 저장 → 복귀. (이후) 메모함 → 프로젝트로 전환.
3. **정리 흐름**: 전체 보기 → 특정 프로젝트 → 태스크 추가/재정렬 → 오늘로 올리기.

## 5. 데이터 모델 (경량)

```
Project
  id, name, color, order, archived_at?, created_at

Task
  id, project_id, title, notes?,
  status: todo | today | done,
  order (프로젝트 내 우선순위),
  today_seq (오늘 목록의 시간순 정렬 키),
  estimate_min?, due_date?,
  rolled_over: bool,
  completed_at?, created_at

Note (빠른 메모 / 인박스 항목)
  id, text, created_at, converted_task_id?
```

- ‘오늘’은 별도 엔티티가 아니라 `status=today`인 태스크의 뷰. **‘지금 하나’는 저장 값이 아니라 파생**: `status=today` 중 미완료이면서 `today_seq`가 가장 앞선 태스크.
- **로컬-우선(local-first)**: 데이터는 브라우저 로컬(IndexedDB 등)에 우선 저장·동작. 클라우드 동기화는 후속.
- 이월은 사용자 로컬 자정 기준 또는 최초 접속 시 처리: 어제의 `today` 미완료 → 오늘 `today` 유지 + `rolled_over=true`.

## 6. 디자인 / UX 원칙 (브랜드 준수)

- **배경 흰색 기본**, 크림·웜그레이는 서피스, 잉크는 텍스트.
- **테라코타는 절제** — 화면당 한두 곳, ‘지금 하나’ 신호에만.
- **그라데이션 금지** — 모든 채움은 단색.
- **위계는 색이 아닌 크기·굵기·여백**으로.
- **문구는 담백하게** — 감성적 위로/재촉 문구 지양(“서두르지 않아도 돼요” 류 금지). 브랜드 보이스의 상한은 “오늘도 한 걸음” 수준.
- 서체: 본문 **Pretendard**, 워드마크 **Outfit**.
- 붉은색·경고색·연체 표시 등 압박 신호 **미사용**.

---

## 7. MVP 범위 요약

**포함**: 프로젝트 CRUD(색상), 태스크 CRUD/정렬/완료, 오늘 화면 + 지금 하나, 빠른 메모 + 메모함, 다정한 이월, 전체 보기, 프로젝트 상세, 반응형 웹.

**제외(후속)**: 하루의 마무리, 검색, 정렬 자동화, 협업, 알림, 연동, 모바일 앱.

---

## 8. 성공 지표 (초기)

- **활성화**: 가입 후 첫 세션에 프로젝트 1개 + 태스크 3개 + ‘지금 하나’ 완료 1회.
- **핵심 습관**: 주 3일 이상 ‘오늘’ 방문, 세션당 지금 하나 완료 ≥ 1.
- **빠른 메모 채택**: 활성 사용자의 40%+가 주 1회 이상 메모 사용.
- **정성 신호**: “압박이 없다/차분하다”는 사용자 피드백.

---

### 결정됨 (Decided)

- **‘지금 하나’ 선정 = 시간순.** `today_seq`가 가장 앞선 미완료 태스크가 자동 선정. (수동 지정·추천 로직 없음)
- **저장소 = 로컬-우선(local-first).** 브라우저 로컬 저장을 기본으로, 클라우드 동기화는 후속 과제.

### 미결 사항 (Open Questions)

1. **⌘N 단축키**: 브라우저 기본 동작과 충돌 → 대체 키/키 조합 확정 필요.
2. **이월 시점 기준**: 로컬 자정 vs 접속 시. 타임존 처리 세부.
3. **프런트 스택**: 로컬-우선 전제하에 프레임워크·로컬 저장 기술(IndexedDB 래퍼 등) 선택.
4. **인증·계정**: 로컬-우선이므로 MVP는 계정 없이 시작 가능 → 클라우드 동기화 도입 시점에 인증 검토.
