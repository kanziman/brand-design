## G-01. 사이드바 (공통)

**목적**: 전역 내비게이션 + 프로젝트 관리 + 빠른 메모 진입.

| 컴포넌트 | 설명 | 동작 |
|----------|------|------|
| G-01-1 로고 | 심볼 + Momento 워드마크 | 클릭 시 오늘(S-01) 이동 |
| G-01-2 내비 | 오늘 / 메모함(개수) / 전체 보기 | 현재 위치 활성 표시(테라코타) |
| G-01-3 프로젝트 목록 | 색 점 + 이름 + 남은 태스크 수 | 클릭 시 S-03 이동 |
| G-01-4 새 프로젝트 | ‘+ 새 프로젝트’ | 인라인 생성(이름·색 선택) |
| G-01-5 빠른 메모 버튼 | ‘+ 빠른 메모 ⌘N’ (하단 고정) | S-05 팝오버 오픈 |

- **반응형**: 좁은 화면(<760px)에서 사이드바는 숨김, 상단 햄버거로 토글.
- **상태**: 프로젝트 0개 → 목록 대신 ‘첫 프로젝트를 만들어 보세요’ 안내.

---

## S-01. 오늘 · 지금 하나

**목적**: 모든 프로젝트를 가로질러 **지금 할 단 하나**에 몰입하게 한다.
**진입**: 앱 기본 진입 화면.

| 컴포넌트 | 설명 | 동작 |
|----------|------|------|
| S-01-1 헤더 | 날짜 + 인사(예: “오늘도 한 걸음.”) | — |
| S-01-2 주간 요약 | “이번 주 N개 끝냈어요” | 압박 수치 아님(완료만 표기) |
| S-01-3 지금 하나 카드 | 프로젝트 태그 + 태스크명 + 예상 소요 | 하이라이트(테라코타) |
| S-01-4 끝냈어요 | 지금 하나 완료 | 완료 처리 → 시간순 다음 태스크 승격 |
| S-01-5 다음 하나 | 지금 것을 뒤로 미룸 | `today_seq` 뒤로 이동, 다음 것 승격 |
| S-01-6 이어서 목록 | 오늘의 나머지 태스크(작게, 시간순) | 항목 클릭 → 완료/편집 |

**‘지금 하나’ 선정 규칙**: `status=today` & 미완료 태스크 중 `today_seq`가 가장 앞선 1건(파생).

**상태**
- **빈 상태**: 오늘 태스크 0개 → “오늘 할 일이 없어요. 프로젝트에서 오늘로 가져오거나, 메모함을 정리해 보세요.” + 바로가기.
- **모두 완료**: 지금 하나 자리에 “오늘 할 일을 다 끝냈어요.” (완료 목록만 하단 표시)
- **이월 항목**: ‘내일로 미룸’ 담백한 라벨. 붉은색·연체 표시 없음.

---

## S-02. 전체 보기

**목적**: 여러 프로젝트를 한눈에 조망하되, 각 프로젝트의 **다음 하나만** 보여 압도감을 줄인다.

| 컴포넌트 | 설명 | 동작 |
|----------|------|------|
| S-02-1 헤더 | “N개의 프로젝트” | — |
| S-02-2 프로젝트 카드 | 색 점 + 이름 | 클릭 시 S-03 이동 |
| S-02-3 다음 하나 | 해당 프로젝트의 시간순 다음 태스크 | 라벨 ‘다음 하나’ |
| S-02-4 진행 바 | 남은 개수 + 차분한 진행 바(단색) | %가 아닌 “N개 남음” |

**상태**
- **다음 하나 없음**: “예정된 일 없음” (댐덤히).
- **프로젝트 0개**: S-06 온보딩으로 안내.

---

## S-03. 프로젝트 상세

**목적**: 한 프로젝트의 태스크를 관리(추가·정렬·완료·오늘로 올리기).

| 컴포넌트 | 설명 | 동작 |
|----------|------|------|
| S-03-1 헤더 | 색 점 + 프로젝트명 + 편집(이름·색·보관) | — |
| S-03-2 태스크 추가 | 한 줄 입력 | Enter로 추가(하단/시간순) |
| S-03-3 태스크 목록 | 우선순위 정렬, 체크·제목·예상소요 | 체크 완료, 드래그 재정렬 |
| S-03-4 오늘로 | 태스크를 오늘(S-01)로 | `status=today` + `today_seq` 부여 |
| S-03-5 완료 섹션 | 완료 태스크 접힘 | 펼치기/되돌리기 |

**상태**
- **빈 상태**: “첫 할 일을 적어 보세요.” 입력에 포커스.
- **보관(archive)**: 목록에서 숨김(삭제 아님).

---

## S-04. 메모함 (Inbox)

**목적**: 빠른 메모로 모인 미분류 항목을 프로젝트 태스크로 정리한다.

| 컴포넌트 | 설명 | 동작 |
|----------|------|------|
| S-04-1 헤더 | “메모함 · N개” | — |
| S-04-2 메모 항목 | 텍스트 + 작성일 | — |
| S-04-3 프로젝트로 전환 | 프로젝트 선택 → 태스크 생성 | `Note.converted_task_id` 기록 |
| S-04-4 삭제 | 메모 제거 | — |

**상태**
- **빈 상태**: “비어 있어요. 떠오른 생각은 ⌘N으로 담아두세요.”

---

## S-05. 빠른 메모 팝오버

**목적**: 몰입을 끊지 않고 떠오른 생각을 한 줄로 담는다.
**진입**: ⌘N(전역) / 사이드바 버튼.

| 컴포넌트 | 설명 | 동작 |
|----------|------|------|
| S-05-1 입력창 | 한 줄 텍스트, 자동 포커스 | 타이핑 |
| S-05-2 저장 | Enter | 메모함에 추가 후 즉시 닫힘 → 원래 작업 복귀 |
| S-05-3 닫기 | Esc / 바깥 클릭 | 저장 없이 닫힘 |

- **원칙**: 프로젝트 지정·분류 UI 없음(마찰 최소화). 연속 입력 지원(저장 후 계속 열어두기 옵션은 후속).
- **주의**: ⌘N 브라우저 충돌 → 대체 단축키 확정 필요(PRD 미결 #1).

---

## S-06. 온보딩 · 빈 상태

**목적**: 최초 실행 시 첫 프로젝트·첫 태스크로 자연스럽게 유도.

| 단계 | 내용 |
|------|------|
| 1 | 간단 소개 1문장 + ‘첫 프로젝트 만들기’ |
| 2 | 프로젝트 이름·색 선택 |
| 3 | 첫 태스크 3개 입력 유도 → 하나를 ‘오늘’로 |
| 완료 | S-01(오늘)로 진입, 지금 하나 노출 |

- 로컬-우선이므로 **계정·로그인 없이 즉시 시작**.

---

## 공통 규칙 요약

- **색/포인트**: 테라코타는 ‘지금 하나’ 등 핵심 신호에만. 나머지는 잉크·웜그레이.
- **압박 신호 금지**: 붉은 배지, 연체 카운트, 달성률 % 미사용.
- **문구**: 담백·사실 위주. 감성 위로 과다 금지.
- **접근성**: 명도 대비 확보, 키보드 내비(⌘N·Enter·Esc), 포커스 표시.
- **반응형**: 데스크톱 우선, 태블릿·모바일 폭까지 레이아웃 붕괴 없이 대응.
