# Momento 랜딩페이지 — PRD

- **버전**: v1 (초안)
- **작성일**: 2026-07-12
- **상태**: 리뷰 대기 (구현 전)
- **관련 문서**: [앱 PRD](./PRD.md) · [화면정의서(ui-spec)](./ui-spec.md) · Momento Design System(토큰·컴포넌트)
- **연결 대상**: 실행 가능한 앱 [`../momento/index.html`](../momento/index.html)

> 목적: 처음 방문한 사람이 **5초 안에 "이건 나를 재촉하지 않는 조용한 작업 공간"임을 이해**하고, 마찰 없이 앱을 열게 한다. 랜딩은 설득이 아니라 **차분한 초대**다.

---

## 1. 개요

Momento 웹앱을 소개하고 실행으로 연결하는 **단일 페이지(one-page) 마케팅 랜딩**.
과장·재촉 없이 제품 철학을 그대로 체감시키는 것이 핵심 — 랜딩 자체가 브랜드의 첫 데모다.

- **한 줄 정의**: "내 일의 순간들을, 조용히 정리하다"를 그대로 보여주는 조용한 소개 페이지
- **타깃**: 여러 건을 동시에 굴리는 1인 사업자·프리랜서·개인 생산성 사용자 (앱 타깃과 동일)
- **톤**: 담백·절제. 감성 위로/긴박함 금지. 상한은 "오늘도 한 걸음" 수준.

---

## 2. 목표 & 비목표

### 목표 (Goals)
1. **핵심 가치 즉시 전달** — "여러 프로젝트, 그러나 지금의 답은 언제나 하나."
2. **마찰 없는 앱 진입** — 어느 위치에서든 한 번의 클릭으로 앱을 연다.
3. **철학 공감** — 왜 붉은 배지·연체 카운트가 없는지(압박 없음)를 이해시킨다.
4. **핵심 기능 이해** — 지금 하나 / 빠른 메모 / 다정한 이월 / 전체 보기를 짧게 보여준다.
5. **의문 해소** — 로그인·저장·가격 등 진입을 막는 질문을 FAQ로 미리 없앤다.

### 비목표 (Non-Goals)
- 회원가입 폼·이메일 수집·뉴스레터 게이트 (진입 마찰 금지)
- 애니메이션 과잉, 패럴랙스, 자동재생 영상
- 경쟁사 비교표/공격적 세일즈 카피
- 가격 결제 플로우 (MVP는 무료·로컬-우선, 가격은 FAQ 한 줄로만)
- 블로그·문서·커뮤니티 등 부가 페이지 (후속)

## 3. 페이지 구조 (섹션 맵)

세로 스크롤 단일 페이지. 순서와 각 섹션의 역할:

| # | 섹션 | 역할 | 필수 요소 |
|---|------|------|-----------|
| 0 | 상단 바 | 워드마크 + 진입 | Momento 워드마크, `앱 열기` 보조 CTA |
| 1 | **히어로** | 가치 즉시 전달 + 1차 CTA | 헤드라인, 서브카피, **주 CTA**, 앱 미리보기 |
| 2 | **철학** | 왜 조용한가 | "압박 없음" 원칙 3가지, 브랜드 한 줄 |
| 3 | **핵심 기능** | 무엇을 하는가 | 기능 카드 4종 |
| 4 | **작동 방식(선택)** | 어떻게 쓰나 | 3스텝 흐름(오늘→하나→끝냈어요) |
| 5 | **FAQ** | 의문 해소 | 접이식 질문 6종 |
| 6 | **마무리 CTA** | 마지막 초대 | 담백한 반복 CTA + 로컬-우선 안내 |
| 7 | 푸터 | 정보 | 워드마크, 저작권, 링크 최소 |

## 4. 섹션별 콘텐츠 상세

### 4.0 상단 바
- 좌: Outfit 워드마크 "Momento". 우: 보조 CTA `앱 열기`(ghost 톤).
- 스크롤해도 방해되지 않게 — 고정하지 않거나, 고정 시 얇고 조용하게.

### 4.1 히어로 (Hero)
- **헤드라인(안)**: "여러 프로젝트, 지금의 답은 하나." — 크게(2xl~3xl), 세미볼드.
- **서브카피(안)**: "혼자 일하는 사람을 위한 조용한 작업 공간. 재촉하지 않고, 지금 할 하나만 건넵니다."
- **주 CTA**: `Momento 열기` (primary, 테라코타). 클릭 → 앱(`../momento/index.html`) 진입.
- **보조 링크**: "핵심 기능 보기 ↓" (앵커 스크롤).
- **비주얼**: 앱의 '오늘' 화면(지금 하나 카드) 실제 스크린샷/목업. 그라데이션·그림자 과잉 금지, 카드 하나만 담백하게.
- CTA는 화면당 하나의 강조만 — 테라코타는 주 CTA에만.

### 4.2 철학 (Core Philosophy)
- 섹션 헤드라인(안): "재촉하지 않는 도구."
- 브랜드 한 줄: **"내 일의 순간들을, 조용히 정리하다."**
- **원칙 3가지**(아이콘 없이 텍스트 카드로도 충분):
  1. **압박 신호 없음** — 붉은 배지도, "N일 지연"도, 달성률 %도 없습니다.
  2. **지금 하나** — 넘치는 목록 대신, 시간순으로 지금 할 하나만.
  3. **다정한 이월** — 못 끝낸 일은 조용히 내일로. 죄책감 없이.

### 4.3 핵심 기능 (Features) — 카드 4종
앱 PRD 3장과 1:1 매핑. 각 카드: 제목 + 한 줄 설명(담백).

| 기능 | 제목(안) | 설명(안) |
|------|----------|----------|
| One Now | 지금 하나 | 모든 프로젝트를 가로질러, 지금 할 단 하나를 크게 건넵니다. |
| Quick Note | 빠른 메모 | 떠오른 생각은 ⌘N으로 한 줄. 흐름을 끊지 않아요. |
| Gentle Rollover | 다정한 이월 | 못 끝낸 일은 조용히 내일로. 붉은 표시는 없습니다. |
| Overview | 전체 보기 | 프로젝트마다 '다음 하나'와 남은 개수만. 압도되지 않게. |

- 보조로 언급(작게): 할 일 상세(메모·링크·마감일), 선택적 로그인(기기 간 동기화).

### 4.4 작동 방식 (How it works) — 선택 섹션
- 3스텝: **① 오늘을 연다 → ② 지금 하나를 한다 → ③ ✓ 끝냈어요.**
- 각 스텝 짧은 캡션 + 작은 화면 조각. 과한 데모 애니메이션 금지.

### 4.5 자주 묻는 질문 (FAQ) — 접이식 6종
초안 질문/답변(담백 톤):

1. **회원가입이 필요한가요?**
   아니요. 계정 없이 바로 시작할 수 있어요. 로그인은 기기 간 동기화를 위한 선택이에요.
2. **내 데이터는 어디에 저장되나요?**
   기본은 브라우저 로컬(local-first)에 저장돼요. 로그인하면 기기 간에 이어서 정리할 수 있어요.
3. **왜 마감일에 빨간 표시나 지연 알림이 없나요?**
   재촉은 몰입을 깨기 때문이에요. 마감일은 보여주되 압박 신호로 쓰지 않아요.
4. **팀이나 협업 기능이 있나요?**
   Momento는 '혼자 일하는 사람'을 위한 도구예요. 초대·댓글·할당 같은 협업 기능은 없어요.
5. **얼마인가요?**
   MVP는 무료예요. (유료 정책은 추후 안내)
6. **모바일에서도 되나요?**
   반응형 웹으로 좁은 화면까지 대응해요. 별도 네이티브 앱은 아직 없어요.

- 컴포넌트: 질문 클릭 시 답 펼침(단순 accordion). 기본은 모두 접힘.

### 4.6 마무리 CTA
- 헤드라인(안): "오늘도 한 걸음."
- 주 CTA `Momento 열기` 반복 + 한 줄: "계정 없이도 바로 시작할 수 있어요."

### 4.7 푸터
- 워드마크 + 저작권 + 최소 링크(앱 열기). 소셜/뉴스레터 없음.

## 5. CTA & 인터랙션

- **주 CTA 라벨(확정 후보)**: "Momento 열기". 동작: 같은 탭에서 앱(`../momento/index.html`)으로 이동.
- **CTA 배치**: 히어로(1차) · 마무리(2차) · 상단 바(보조). 페이지당 강조 CTA는 시야에 하나만.
- **앵커 내비**: "핵심 기능 ↓" 등 부드러운 스크롤. 필수 아님.
- **FAQ**: 클릭 토글, 키보드 접근(Enter/Space), 포커스 표시.
- **모션**: `--ease-calm` 120~200ms 이내의 절제된 전환만. 스프링·바운스·루프·패럴랙스 금지.

## 6. 디자인 / 브랜드 준수

- **토큰 재사용**: Momento Design System(`../momento/tokens/*`)의 색·타이포·간격·라운드 그대로.
- **색**: 흰 배경 기본, 크림·웜그레이는 서피스. **테라코타는 주 CTA와 '지금 하나' 강조에만.**
- **그라데이션·그림자 과잉·패턴/텍스처 금지.** 채움은 단색, 카드는 1px 웜그레이 보더.
- **타이포**: 본문 Pretendard, 워드마크 Outfit. 위계는 크기·굵기·여백으로.
- **여백은 기능** — 섹션 사이 넉넉한 호흡. 한 화면에 너무 많은 것을 담지 않는다.
- **카피 규칙**: 사실 위주·담백. 느낌표 남발·감성 위로("서두르지 않아도 돼요" 류) 금지. 이모지 장식 금지.
- **접근성**: 명도 대비 확보, 키보드 내비, 포커스 표시, 이미지 alt.

---

## 7. 반응형

- 데스크톱 우선, 태블릿·모바일 폭까지 레이아웃 붕괴 없이 대응.
- 기능 카드 그리드: 데스크톱 4열(또는 2×2) → 모바일 1열.
- 히어로: 좌우 2단(카피 | 미리보기) → 모바일 세로 스택.
- 상단 바: 모바일에서 워드마크 + CTA만 유지(햄버거 불필요, 링크 최소).

## 8. 성공 지표 (초기)

- **핵심 전환**: 방문 → '앱 열기' 클릭률(CTA CTR).
- **이해도**: 히어로만 보고 이탈하지 않고 최소 1개 섹션 이상 스크롤한 비율.
- **의문 해소**: FAQ 펼침 후 CTA 클릭으로 이어진 비율.
- **정성 신호**: "차분하다 / 압박이 없다"는 첫인상 피드백.

---

## 9. 범위 & 산출물

**포함(MVP)**: 단일 페이지(상단 바·히어로·철학·기능·FAQ·마무리 CTA·푸터), 앱으로의 CTA, 반응형, 접근성 기본, 디자인 토큰 재사용.

**제외(후속)**: 다국어(i18n), 애널리틱스 이벤트 배선, A/B 카피 실험, 블로그/문서, OG·메타 이미지 세트(초기엔 기본만), 실제 앱 스크린샷 자동 갱신.

## 10. 열린 질문 (Open)

1. **배포 위치/도메인** — 앱과 같은 정적 호스팅에 둘지, CTA 링크를 상대경로(`../momento/`)로 둘지 절대 URL로 둘지.
2. **주 CTA 문구 확정** — "Momento 열기" vs "지금 시작하기" vs "오늘 열기".
3. **히어로 비주얼** — 실제 앱 스크린샷 캡처 사용 vs 정적 목업 재현.
4. **가격 메시지** — 무료 유지 기간/유료 계획을 FAQ에 어디까지 명시할지.
5. **작동 방식 섹션 포함 여부** — 4번 섹션을 넣을지(길이·집중도 트레이드오프).
