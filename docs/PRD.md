## 1. 개요

**Momento**는 1인·프리랜서를 위한 미니멀 프로젝트 관리 웹앱이다.
여러 프로젝트를 관리하되, 화면은 늘 **'지금 집중할 단 하나'**를 건네 압도감을 없앤다.

- **한 줄 정의**: 여러 프로젝트를 다루지만, 지금의 답은 언제나 하나인 조용한 작업 공간
- **타깃**: 여러 건을 동시에 굴리는 1인 사업자·프리랜서·개인 생산성 사용자
- **경쟁 대비**: Jira/Asana의 협업·기능 과잉 ✗ → 개인 몰입·절제 ✓

### 해결하는 문제
혼자 여러 프로젝트를 굴리는 사람의 진짜 고통은 '할 일이 많다'보다 **'도구가 나를 재촉하고 압도한다'**이다. 밀린 항목의 붉은 배지, 넘치는 목록, 끊기는 몰입이 문제다.

---

## 2. 목표 & 비목표

### 목표 (Goals)
1. 여러 프로젝트를 가로질러 **지금 할 하나**를 명확히 제시한다.
2. 떠오른 생각을 **흐름을 끊지 않고** 담게 한다 (빠른 메모).
3. 못 끝낸 일이 **압박이 아니라 조용한 이월**로 이어지게 한다.
4. 각 할 일에 **맥락(메모·링크·마감일)**을 가볍게 붙여, 다시 잡을 때 헤매지 않게 한다. `NEW v2`
5. 모든 화면이 브랜드 철학(여백·절제·재촉하지 않음)을 체감시킨다.

### 비목표 (Non-Goals, MVP 제외)
- 팀 협업(멤버 초대·댓글·할당·공유)
- 푸시/이메일 알림, 리마인더 스팸
- 화면·웹 스크린샷 캡처, 파일/이미지 클리핑 (텍스트 메모 + URL 링크만)
- 간트·타임트래킹·리포트 대시보드
- 네이티브 모바일 앱 (반응형 웹까지만)
- 서드파티 연동(캘린더·슬랙 등) — 로그인은 **동기화 목적의 인증만**, 데이터 연동은 아님

## 3. 핵심 기능 (MVP)

### 3.1 지금 하나 (One Now) — 킬러 피처
- '오늘' 화면 상단에 **단 하나의 태스크**를 크게 표시.
- **NEW v2 — 시각 처리 확정**: 지금 하나 카드는 **테라코타 보더(1.5px) + `--accent-soft` 배경 + "지금 하나" 라벨**로 표현한다. 테라코타를 '지금 하나' 신호에만 쓰는 브랜드 규칙을 카드 단위로 고정.
- **선정 방식(확정): 시간순.** `status=today` 미완료 중 `today_seq`가 가장 앞선 태스크가 자동으로 '지금 하나'가 된다. 수동 지정 없음.
- 액션: **[✓ 끝냈어요]** → 완료 처리 후 다음 태스크 승격 · **[다음 하나 →]** → `today_seq`를 뒤로 보내 미루기.
- 나머지 오늘 태스크는 하단 **'이어서'**에 담백하게 나열. `NEW v2` — 각 행에 체크박스·예상 소요·이월 라벨을 표시하고, 제목 클릭 시 상세 드로어(S-07)로 진입.
- **NEW v2 — 오늘 상태 3종**:
  - 지금 하나 있음(기본)
  - **모두 완료**: "오늘 할 일을 다 끝냈어요." 카드 + '오늘 완료 · N' 목록
  - **빈 상태**: 안내 문구 + [전체 보기 →]/[메모함 →] 바로가기

### 3.2 다중 프로젝트 관리
- 프로젝트 CRUD, 각 프로젝트에 **식별 색상**(테라코타/세이지/블루/플럼 뮤트 팔레트) 지정.
- 사이드바에 프로젝트 목록 + 남은 태스크 수.
- **전체 보기**: 프로젝트 카드 그리드, 카드마다 **'다음 하나'**와 진행 바(남은 개수)만 노출.
- **NEW v2 — '다음 하나' 정렬**: `today`(오늘로 올라간 것) 우선, 그다음 `today_seq`/순서. 예정된 일 없으면 "예정된 일 없음"(담백).

### 3.3 빠른 메모 (Quick Note) & 메모함 (Inbox)
- 전역 단축키 **⌘N**(Cmd/Ctrl+N)으로 팝오버 → 한 줄 입력 → 저장 후 하던 일로 복귀.
- **NEW v2 — 단축키 충돌 처리**: 입력 필드(INPUT/TEXTAREA)에 포커스가 있을 땐 ⌘N을 가로채지 않는다(브라우저 기본 동작 존중). 팝오버가 열려 있으면 Enter=저장, Esc=닫기.
- 메모는 프로젝트 지정 없이 **메모함(Inbox)**에 모임.
- **NEW v2 — 인라인 전환(triage)**: 메모함에서 '프로젝트로 전환' → **같은 카드 안에서** 프로젝트 색점 칩을 골라 태스크로 이관(별도 드롭다운/모달 없음). '취소'로 되돌림. 작성 시간은 상대 표기("방금/오늘/어제/N일 전").

### 3.4 할 일 상세 (Task Detail) `NEW v2`
- 태스크 제목 클릭 시 **우측 슬라이드오버 드로어**로 상세를 연다.
- 구성: 제목(편집), 완료 체크박스, **프로젝트 / 예상 소요 / 마감일 / 상태**(오늘 or '오늘로 ↑'), **링크 목록**(URL 여러 개 추가·삭제), **작업 메모**(멀티라인), '이 할 일 삭제'.
- 닫기: Esc · 바깥 스크림 클릭 · 닫기 버튼.

### 3.5 다정한 이월 (Gentle Rollover)
- 오늘 못 끝낸 태스크는 다음 날 오늘로 **조용히 이월**. 붉은 배지·"N일 지연"·연체 카운트 **없음**.
- 표시는 담백하게 "내일로 미룸" 라벨(외곽선 pill, 색 신호 아님).

### 3.6 선택적 로그인 · 동기화 (Optional Auth) `NEW v2`
- 앱 진입 시 **로그인 게이트**: [Google로 계속하기] 또는 **[계정 없이 시작하기]**.
- **로컬-우선 유지**: 계정 없이도 로컬에 저장·동작. 로그인은 **기기 간 동기화용 선택**이며 강제하지 않는다.
- 사이드바 하단 **계정 행(AccountRow)**: 로그인 시 아바타+이메일+로그아웃 / 로컬 모드 시 세이지 점 + "로컬에 저장 중 · 로그인".
- 카피는 담백 톤 유지("로그인하면 어느 기기에서든 이어서 정리할 수 있어요").
- ⚠️ Google 'G' 마크는 트레이드마크 회피용 임시 플레이스홀더 — 공식 브랜드 SVG로 교체 필요.

## 4. 주요 화면 & 흐름

| 화면 | 목적 | 핵심 요소 |
|---|---|---|
| **로그인 게이트** `NEW v2` | 로컬/동기화 선택 | Google · 계정 없이 시작 |
| **오늘** | 지금 하나에 몰입 | 지금 하나 카드, 이어서, 오늘 완료, 주간 완료 수 |
| **전체 보기** | 프로젝트 조망 | 프로젝트 카드(다음 하나 + 진행 바) |
| **프로젝트 상세** | 한 프로젝트의 태스크 관리 | 태스크 목록, 추가/완료/오늘로 |
| **메모함** | 빠른 메모 정리 | 메모 카드 → 인라인 프로젝트 전환 |
| **빠른 메모 팝오버** | 마찰 없는 입력 | 한 줄 입력 + 저장 |
| **할 일 상세 드로어** `NEW v2` | 맥락 붙이기 | 메모·링크·마감일·상태 |

### 대표 흐름
1. **몰입 흐름**: 오늘 → 지금 하나 수행 → ✓ 끝냈어요 → 다음 하나 자동 노출.
2. **메모 흐름**: 작업 중 아이디어 → ⌘N → 한 줄 저장 → 복귀. (이후) 메모함 → 인라인 프로젝트 전환.
3. **정리 흐름**: 전체 보기 → 특정 프로젝트 → 태스크 추가/오늘로 올리기.
4. **맥락 흐름** `NEW v2`: 태스크 제목 클릭 → 상세 드로어 → 링크·메모·마감일 기록.

## 5. 데이터 모델 (경량)

```
Project
  id, name, color, order?, created_at?

Task
  id, project_id, title,
  status: todo | today | done,
  done: bool,                       # 완료 플래그(상태와 함께 유지)
  today_seq,                        # 오늘 목록의 시간순 정렬 키
  rolled_over: bool,
  estimate,                         # 예상 소요(자유 텍스트, 예 "30분")   NEW v2: 확정
  due,                              # 마감일(자유 텍스트 표기)             NEW v2
  note,                             # 작업 메모(멀티라인)                  NEW v2
  links: string[]                   # 참고 링크 URL 배열                    NEW v2

Note (빠른 메모 / 인박스 항목)
  id, text, ts (작성 시각)

App/Auth  NEW v2
  weekly_done                       # 이번 주 완료 수(주간 요약)
  auth: { authed: bool, mode: 'google' | 'local' }
```

- '오늘'은 별도 엔티티가 아니라 `status=today`인 태스크의 뷰. **'지금 하나'는 파생값**(`status=today` & 미완료 & `today_seq` 최솟값).
- **로컬-우선(local-first)**: 데이터는 브라우저 로컬에 우선 저장·동작.
  - 프로토타입 저장 키: 데이터 `momento-hifi-v2`, 인증 `momento-auth-v2` (localStorage).
- 이월: 최초 접속 시 어제의 `today` 미완료 → 오늘 `today` 유지 + `rolled_over=true`.

## 6. 디자인 / UX 원칙 (브랜드 준수)

- **배경 흰색 기본**, 크림·웜그레이는 서피스, 잉크는 텍스트.
- **테라코타는 절제** — 화면당 한두 곳, '지금 하나' 신호에만. (v2: 지금 하나 카드 보더+`accent-soft`, '오늘로 ↑' pill)
- **그라데이션 금지** — 모든 채움은 단색.
- **위계는 색이 아닌 크기·굵기·여백**으로.
- **문구는 담백하게** — 감성적 위로/재촉 문구 지양. 상한은 "오늘도 한 걸음" 수준.
- **압박 신호 미사용** — 붉은색·경고색·"D-n"·연체 카운트·달성률 % 금지. 마감일도 지난 날짜에 색을 바꾸지 않는다(담백한 `--text-secondary`).
- 서체: 본문 **Pretendard**, 워드마크 **Outfit**.
- 오버레이는 스크림 `rgba(26,26,26,0.15)`(블러 없음). 중앙 팝오버(빠른 메모)와 우측 슬라이드오버(상세 드로어) 두 패턴.
- **반응형**: `<760px`에서 사이드바 → 상단 햄버거 + 좌측 오프캔버스 드로어, 전체 보기 그리드 1열. `NEW v2`

## 7. MVP 범위 요약

**포함**: 프로젝트 CRUD(색상), 태스크 CRUD/완료/오늘로, 오늘 화면 + 지금 하나(+완료/빈 상태), 빠른 메모 + 메모함(인라인 전환), 할 일 상세(메모·링크·마감일), 다정한 이월, 전체 보기, 프로젝트 상세, 선택적 로그인·계정 행, 반응형 웹, 로컬 저장.

**제외(후속)**: 실제 클라우드 동기화 백엔드, 하루의 마무리, 검색, 정렬 자동화, 협업, 알림, 서드파티 연동, 모바일 앱, 아이콘셋 정식 도입.

## 8. 성공 지표 (초기)

- **활성화**: 첫 세션에 프로젝트 1개 + 태스크 3개 + '지금 하나' 완료 1회.
- **핵심 습관**: 주 3일 이상 '오늘' 방문, 세션당 지금 하나 완료 ≥ 1.
- **빠른 메모 채택**: 활성 사용자의 40%+가 주 1회 이상 메모 사용.
- **정성 신호**: "압박이 없다/차분하다"는 사용자 피드백.

## 9. 결정 & 남은 과제

### 결정됨 (Decided)
- **'지금 하나' 선정 = 시간순** (`today_seq` 최솟값 미완료 태스크).
- **저장소 = 로컬-우선.** localStorage 프로토타입 검증 완료, 후속 클라우드 동기화.
- **로그인 = 선택.** 계정 없이 시작 가능, Google 로그인은 동기화용 옵션. `NEW v2`
- **⌘N = 입력 필드 밖에서만 가로챔.** `NEW v2`
- **할 일 상세 필드 = 예상 소요 · 마감일 · 링크[] · 메모.** `NEW v2`

### 남은 과제 (Open)
1. **아이콘셋**: 링크/닫기/마감일/햄버거 등 기능 글리프 증가 → Lucide 등 thin-stroke 세트 정식 도입 여부(현재 🔗/✕/📅는 임시 유니코드).
2. **Google 로고 자산**: 공식 마크 SVG 반입(현재 플레이스홀더).
3. **클라우드 동기화 백엔드**: 인증·충돌 해결·오프라인 병합 설계.
4. **마감일 입력 방식**: 현재 자유 텍스트 → 데이트피커 도입 여부(압박 신호 없는 형태로).
5. **이월 시점 기준**: 로컬 자정 vs 접속 시, 타임존 처리.
